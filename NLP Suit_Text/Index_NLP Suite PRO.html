PRO vs Light Version

The application operates in two modes. "Light" mode provides quick access to standard models, while "PRO" mode unlocks the full suite of customization and fine-tuning capabilities.

Light Mode

  • ‚úì Access to all core NLP tasks.
  • ‚úì Uses pre-trained, general-purpose models (like `distilgpt2`).
  • ‚úì Instant results for quick analysis.
  • ‚úó Cannot upload custom data.
  • ‚úó Cannot fine-tune models.
  • ‚úó No preprocessing pipeline.

PRO Mode

  • ‚úì All core NLP tasks.
  • ‚úì Corpus-Aware Model Fine-Tuning.
  • ‚úì Upload custom corpus (.txt, .csv, or images via OCR).
  • ‚úì Selectable base models (`distilgpt2` or `gpt2`).
  • ‚úì Interactive preprocessing pipeline (slang, emojis, spelling, etc.).
  • ‚úì Robust, long-duration training with checkpoint saving.

The Development Journey

Phase 1: v2.6 - v2.7

Early Scaffolding: A Usable Application

The initial versions were functionally sound but lacked a professional user experience. The primary focus was to move beyond a basic script and build a robust, interactive GUI.

A critical step was making the Word Cloud generation non-blocking using Python's `threading` module. Previously, the main app would freeze. A "Home" button was added to avoid forcing users to restart, and the UI was re-architected into a fully event-driven, non-blocking model.

Phase 2: v2.8 - v4.0

The "PRO" Vision: Customization & Control

With the UI stable, the goal shifted to making "PRO" mode truly powerful. The fine-tuning logic was enhanced from a naive line-based check to a robust, token-based chunking method‚Äîa standard in production NLP.

This release (dubbed the "Dr. Aryan Verma" Edition) fully realized the vision. It introduced the interactive preprocessing pipeline, letting users toggle cleaning steps. It also added the choice between `distilgpt2` (fast) and `gpt2` (robust). Critically, automatic checkpoint saving after each epoch was implemented, a feature essential for any serious deep learning app.

Phase 3: v5.0 - v6.4

The Bug Hunt: Stabilization & Polish

As features grew, so did the complexity‚Äîand the bugs. This phase was a humbling but necessary period of intense bug fixing and performance optimization, driven by user feedback.

A major performance bottleneck emerged: the application would freeze during preprocessing. The culprit was a highly inefficient `O(n^2)` spelling correction algorithm. This was re-engineered to first identify all unique misspelled words in a single pass (`O(n)`) and then apply corrections, resolving the freeze.

Phase 4: v7.0 - v7.3

The Final Polish: "Arcade" Theme & Stability

With core functionality stable, the final push was to achieve an expert-level GUI and squash the last remaining bugs. The final "Arcade" theme‚Äîa high-contrast, neon-on-dark interface‚Äîwas introduced.

This, however, introduced new `TclError` crashes related to custom styling. The last set of bugs were the most frustrating: a configuration mismatch in `TrainingArguments` crashed fine-tuning, and text was disappearing on hover. These were meticulously traced and fixed, resulting in a stable, powerful, and visually compelling final product.

Critical Problems & Resolutions

Performance Fix: O(n¬≤) Spelling Correction Freeze (v6.4) Click to expand

Problem:

The application would completely freeze when running spelling correction on large text corpora. The algorithm was inefficiently re-checking the entire corpus for every single word, a classic O(n¬≤) complexity problem.

Resolution:

The pipeline was re-engineered. The new logic first identifies all *unique* misspelled words in a single pass (O(n)), creates a correction map, and then iterates one more time to apply the corrections. This resolved the freeze entirely and provided a better UX by splitting the progress bar into "Identifying" and "Applying".

GUI Crash: TclError on "Arcade" Theme (v7.2) Click to expand

Problem:

Introducing the new "Arcade" theme caused the application to crash on startup with a `TclError: Layout Custom.TLabelFrame not found`. The custom Tkinter (ttk) styles were not being defined correctly before being applied.

Resolution:

After careful debugging, the issue was isolated to an improper style definition. The fix involved ensuring that all custom styles were correctly defined and applied, guaranteeing the application would start reliably with the new theme.

Model Crash: Transformers TrainingArguments Conflict (v7.3) Click to expand

Problem:

The fine-tuning process, the core of PRO mode, was crashing. This was traced to a subtle configuration conflict in the Hugging Face `TrainingArguments` class, specifically when `load_best_model_at_end` was enabled without providing a separate evaluation set.

Resolution:

The configuration was corrected to align with the library's requirements, resolving the training crash and stabilizing the fine-tuning feature.

UI Bug: Invisible Text on Hover (v7.3) Click to expand

Problem:

A frustrating visual bug persisted where text on buttons, radio buttons, and dropdown menus would become invisible (or white-on-white) when hovered over, making the app unusable.

Resolution:

This required a meticulous review of all custom `ttk` style maps. The issue was a missing or incorrect foreground color setting for the `active` (hover) state. This was permanently resolved by explicitly defining the correct text color for all widget states.

Code Deep-Dive

This core function from `Streamlit_NLP.py` shows the advanced, multi-stage preprocessing pipeline, complete with weighted progress reporting. This function is the engine behind the "PRO Mode" corpus cleaning. 


# This function is the heart of the "PRO Mode" preprocessing pipeline
# It's designed to be modular and provide granular feedback to the UI

CHAT_WORDS = {
    "lol": "laughing out loud", "brb": "be right back", "btw": "by the way",
    "omg": "oh my god", "imo": "in my opinion", "imho": "in my humble opinion",
    # ... many more
}

def preprocess_corpus(text, options, progress_callback=None):
    if not isinstance(text, str): return ""
    
    # Each stage is weighted to provide a more accurate ETA on the progress bar
    stages = {
        "Cleaning HTML/URLs": (10, lambda t: BeautifulSoup(t, "html.parser").get_text() and re.sub(r'httpsK?://\S+|www\.\S+', '', t) if options.get("Remove HTML/URLs") else t),
        "Handling Emojis": (5, lambda t: emoji.demojize(t, delimiters=(" ", " ")) if options.get("Handle Emojis") == "Convert to Text" else (emoji.replace_emoji(t, replace='') if options.get("Handle Emojis") == "Remove" else t)),
        "Expanding Chat Slang": (10, lambda t: ' '.join([CHAT_WORDS.get(word.lower(), word) for word in t.split()]) if options.get("Expand Chat Slang") else t),
        "Lowercasing": (5, lambda t: t.lower() if options.get("Lowercase") else t),
        "Tokenizing": (15, lambda t: word_tokenize(t)),
        "Identifying Misspelled Words": (30, lambda t: (t, spell.unknown(t)) if options.get("Spelling Correction") and SPELLCHECK_AVAILABLE and isinstance(t, list) and (spell := SpellChecker()) else (t, set())),
        "Applying Corrections": (15, lambda t_tuple: ([spell.correction(token) if token in t_tuple[1] else token for token in t_tuple[0]] if options.get("Spelling Correction") and SPELLCHECK_AVAILABLE and isinstance(t_tuple[0], list) and (spell := SpellChecker()) else t_tuple[0])),
        "Finalizing Tokens": (10, lambda t: ' '.join(filter_and_lemmatize(t, options)) if isinstance(t, list) else t)
    }

    total_weight = sum(w for w, f in stages.values())
    cumulative_weight = 0
    
    processed_data = text

    for stage_name, (weight, func) in stages.items():
        if progress_callback:
            # The callback updates the Streamlit/Tkinter progress bar
            progress_callback(cumulative_weight / total_weight, f"Processing: {stage_name}...")
        
        processed_data = func(processed_data)
        cumulative_weight += weight
        
    if progress_callback:
        progress_callback(1.0, "Finished")

    return processed_data


def filter_and_lemmatize(tokens, options):
    final_tokens = []
    lemmatizer = WordNetLemmatizer()
    stop_words = set(stopwords.words('english'))
    
    for token in tokens:
        if token is None: continue
        
        if not token.isalpha():
            if options.get("Remove Punctuation"):
                continue
            else: 
                pass
        
        if options.get("Remove Stopwords") and token.lower() in stop_words:
            continue
            
        if options.get("Lemmatize"):
            final_tokens.append(lemmatizer.lemmatize(token))
        else:
            final_tokens.append(token)
            
    return final_tokens
